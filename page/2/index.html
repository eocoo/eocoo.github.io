NIO Channel & Selector

Channel 与之前的 流Stream 是差不多一个等级的，只不过 Stream 都是单向的，比如InputStream或OutputStream，而 Channel 可以是单向的，也可以是双向的，既可以同时支持读写操作，具体由其实现的接口所决定

可以看到 read 和 write 方法接受的都是一个 ByteBuffer 参数，其中 read 就是 Channel 往 ByteBuffer 中 put 数据，而 write 就是 Channel 从 ByteBuffer 中 get 数据，以便发送给其他远程主机。两者均返回已传输的字节数，可能比缓冲区的字节数少甚至可能为零，缓冲区的position位置也会前移对应的字节数，如果只进行了部分传输，缓冲区可以被重新提交给通道并从上次中断的地方继续传输，该过程可以重复进行直到缓冲区的hasRemaining()方法返回 false 值。
