1. sed

Stream Editor文本流编辑，一个非交互式的面向字符流的编辑器，它逐行处理内容，每次将当前处理的行存储到临时缓冲区（模式空间 pattern space），处理完成后并输出，接着处理下一行，直到文件结束。 sed作为常见的文本处理工具之一，是一个非常有用且重要的命令，也是写自动化shell脚本的基础。

语法

sed [options] 'scripts' inputfile

-options

  • -n ：只打印匹配行

  • -e ：直接在命令行模式上进行sed动作编辑，默认选项

  • -f ：执行指定文件中的sed动作

  • -r ：支持扩展表达式

  • -i ：直接修改文件内容

常见动作

  • a ：在当前行的下面新增一行内容；
  • c ：替换匹配行
  • d ：删除匹配行
  • i ：在当前行的上面新增一行内容；
  • p ：打印
  • s ：替换匹配文本

1.1. 文本替换

sed -i 's/old/new/' m.txt：将每行的第一个old替换为new

sed -i 's/old/new/3' m.txt：将每行的第三个old替换为new

sed -i 's/old/new/g' m.txt：将所有的old替换为new

sed -i '4,7s/^/#/' m.txt：第4-7行注释掉

sed -ri 's/^( )+//' a.txt：删除行首的空格

其中，替换操作里的分割符/可改为其他字符，如#或$，在遇到文件路径替换时会比较方便

1.2. 文本行打印

sed -n '4,7p' m.txt：打印第4-7行

sed -n '4,+10p' m.txt：输出第4行及其下面10行的内容

sed -n 'p;n' m.txt：输出奇数行

sed -n 'n;p' m.txt：输出偶数行

sed -n '$=' m.txt：输出文件的行数

1.3. 文本行添加

sed -i '2iXX' m.txt：在第2行上面添加文本行XX

sed -i '2aXX' m.txt：在第2行下面添加一行文本行XX

sed -i '4,7aXX' m.txt：在第4-7行的每一行下面添加文本行XX

如果是多行文本，可以用换行符\n分割

1.4. 文本行替换

sed -i '2cXX' m.txt：在第2行替换为文本行XX

1.5. 文本行删除

sed -i '3,5d' m.txt：删除第3-5行

sed -i '/xml/d' m.txt：删除所有包含xml的行

sed -i '/xml/!d' m.txt：删除所有不包含xml的行

sed -i '$d' m.txt：删除最后一行

sed -i '/^$/d' m.txt：删除空行

1.6. sed脚本

对于一些复杂的操作可以存为脚本，然后通过-f调用，比如

sed -f test.sed m.txt：执行脚本中的动作

2. awk

awk是一个强大的文本分析工具，相对于sed，awk在对数据分析并生成报告时显得尤为强大，awk也是将文件逐行读入，然后将每行分隔切片再进行各种分析处理。其名称取自三位创始人的首字符（Alfred Aho、Peter Weinberger、Family Name）。

语法

awk [options] 'script' var=value inputfile

-options

  • -F：指定解析分隔符（默认空格），可以是字符串或者正则表达式

  • -v：赋值一个自定义变量

  • -f：从文件中读取awk命令

2.1. 获取字段

awk '{print $2}' m.txt：获取以空格分隔后，每行的第二个字段

awk -F '_' '{print $2}' m.txt：获取以_分隔后，每行的第二个字段
