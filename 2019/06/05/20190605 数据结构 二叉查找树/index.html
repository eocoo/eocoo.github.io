数据结构. 二叉查找树

words: 7.1k     views:

引言

前面介绍线性数据结构时提到，如果数据本身是有序的，那么可以用二分法来降低查找的时间复杂度，但是在线性结构上实现二分法总有这样或那样的局限， 比如链式数据不好定位中间元素而数组总免不了拷贝成本

  • 数据结构. 链表、队列、栈

因此，有前辈提出了树这种结构，它也是一种链式数据结构，但它巧妙地解决了链式数据只能从头或者从尾开始查找的问题，而是从中间开始向两边查找， 不管是下面的查找二叉树，还是后面要说的Avl树、红黑树或者B树，其基本思想都是二分法， 区别只在于通过不同的规则来维持树的平衡，以便使查找的复杂度更接近于二分法。

1. 定义

树：定义树的一种自然的方式是递归，树由根节点及其子树构成，而每一个子树又有自己的根节点与子树。

二叉树：二叉树在树的基础上加了一个限定，对于任意节点，至多只能有两个子树。

二叉查找树：二叉查找树又在二叉树的基础上加了一个限定，对于任意节点X，它的左子树中的所有节点的值都小于X的值，而右子树中的所有节点的值都大于X的值。

相关定义：
阶：节点的子树个数，所以二叉树就是一个二阶树。
叶子节点：没有子树的节点，即阶为零。
节点高度：对任意节点x，其高度为叶子节点到x节点的路径长度。
树的高度就是叶子节点到根节点的高度。
节点深度：对任意节点x，其深度为根节点到x节点的路径长度。
树的深度就是根节点到叶子节点的深度。
满二叉树：所有叶子节点都在同一层，而且其他层任意节点都有两个子节点，从形象上看就像一个等腰三角形。

2. 分析实现

这里主要分析下几个常见的操作，比如查找、插入、删除以及迭代，其他的操作都可以在此基础上进行实现。

2.1. 查找

由于树的特性，我们可以采用递归的方式来实现查找（存在耗尽栈空间的风险，但实际应用中一般不会直接使用二叉树，而是会想办法使树的期望高度为logN）。
如果是空树，则直接返回false，否则从树根开始比较，如果大于就向左递归，否则向右递归，直至找到等于的节点返回true或者最后节点为空返回false为止。

2.2. 插入

首先知道，查找二叉树保存不了重复的数据，如果数据确实重复，只能选择丢弃或者覆盖。在实现插入时，可以先与2.1一样进行查找， 如果找到值相等的节点，就进行覆盖，否则最后结束的位置一定是个空节点，也就是待插入的位置（插入的节点一定为叶子节点）。

2.3. 删除

对于要删除的节点X，考虑几种情形：
1、X是一片树叶，则可以直接删除；
2、X只有一个子节点，则可以让X的父节点指向X的儿子，即绕过X就可以了；
3、X有两个子节点。
根据查找二叉树的特点：任意节点肯定大于它左边的节点，而小于它右边的节点。因此，如果要删除节点X，可以用X左子树的最大节点（称为X的前驱）leftMax代替X， 然后删除leftMax；或者用X右子树的最小节点（称为X的后继）rightMin代替X，然后删除rightMin。这样可以将删除情形转换成1或2，并且删除之后的结果依然满足二叉树的定义。比如：

但这样发生了两次查找过程，可以实现一个removeRigthMin来直接删除并返回最小的节点。另外，如果总是删除右子树中的节点，将会使得树越来越像左倾斜， 因此可以对称地实现一个removeLeftMax，并在删除节点的时候随机地调用它们中的一个，这样来尽量保持树的平衡。
上面是按照《数据结构与算法分析》中思路来说的，它在实现中没有记parent节点，事实上如果在节点中记住parent，那么可以实现一个传参为节点node的删除方法， 这样就可以先找到要删的节点，然后直接删除就可以了。

3. java实现

在Java集合框架中可以看到查找二叉树的应用，比如TreeMap或者TreeSet就是基于红黑树（带有平衡条件的查找二叉树）实现的。因此，我们也可以用查找二叉树来实现一下Set和Map接口， 当然这里不会实现完整的接口，那样也没有意义，只是实现一些主要的操作以便说明问题。

另外有一个想法：如果在查找二叉树的结构上再维护一个链表，那么就可以处理重复数据的问题，而且可以更容易地实现迭代或者范围查询。 这种想法在很多地方都有体现，比如在后面会介绍的B树与B+树，以及java中LinkedHashMap与HashMap的实现。
下面实现Map时将会采用这种想法，实际实现时会扩展一个子接口MultiTreeMap，虽然这里用查找二叉树实现起来并没有使用价值，但后面可以在此基础上利用红黑树实现， 那么在有些场景下就可以弥补TreeMap不能处理的重复元素场景。

3.1. Set

BinarySearchTreeSet中的节点没有记parent，所以在实现迭代器iterator()的时候先用递归将整颗树放入List中，然后把迭代委托给List，这个递归的深度将是元素的总数，很可能会耗尽栈空间， 即便不考虑这个问题，这种实现也是不可取的。
另外，与后面一样，所有关于树的实现都统一提供一个printTree接口，以便能够直观画出树的结构，其实很简单，就是用Map记住每层的节点，然后依次打印。

3.2. MultiTreeMap

MultiBinarySearchTreeMap中的节点记了parent，但它也没有像2.4说的那样实现迭代，因为它可以利用链表进行迭代。另外，它重载了一个构造器，可以设定迭代的顺序以及重复数据是否覆盖。
概括下基本思路：
对于查找，直接在树中进行，如果是范围迭代，首先在树中确定首尾节点，然后通过链表进行遍历。
对于插入，首先在树中进行查找node，如果树中已存在，则直接在链表中链接为node.next；否则如果在树中插入为node.left，则在链表中链接为node.prev; 如果在树中插入为node.right，则在链表中链接为最后一个重复node的next。 类似的对于删除，首先在树中进行查找node，如果没有则直接返回；如果存在且链表中有重复的，则用node.next的值替换node的值，然后删除node.next；否则分别从树和链表中删除node。

4. 查找二叉树的高度

关于节点数为N的二叉查找树的高度H。
假设第n层的节点数记为An，前n层的节点数之和为Sn。

首先如果刚好是一颗满二叉树，那么其在第h层的节点数可以满足等比通项：Ah = 2^(h-1)，
因此求和有：1 + 2 + 2^2 +…+ 2^(H-1) = 2^H - 1 = N，即H = logN - 1
这里引申一下，如果是满M叉树（比如后面介绍的B树，阶为M，每个节点最多包含M-1个元素），
则有：(M-1)(1 + M + M^2 +…+ M^(H-1)) = M^H - 1 = N，即H = logN - 1，底数为M（一般如果不作说明，底数都默认为树的阶）。

但是满二叉树只是最好的情形，二叉树的实际形状依赖于节点插入和删除的动作，试想如果是按顺序插入，那么最差的情形二叉树将会退化成一个线性链表，这样查找复杂度又从logN退化成N， 因此，为了保持二叉树的平衡，需要附加一些平衡条件，我们将带有平衡条件的二叉树称为平衡二叉树，后面会再具体介绍，比如Avl树或者红黑树。
