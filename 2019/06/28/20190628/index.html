借助poi解析Excel 一 实现readLine读取

words: 3.2k    views:    time: 15min
开发笔记
poi

一般读取文本时会使用BufferedReader，它在InputStreamReader的基础上进行了装饰，提供readLine()来简化文本行的读取。使得行数据的读取可以像流水线一样，每当读取完一行后，它就自动准备好下一行并等待获取，直至全部读取结束。所以这里也希望能够在poi的基础上提供一个类似readLine()的接口来读取Excel，并可以比较方便地获取行数据中的字段。

1. csv文本解析

可以先定义一个统一的读取接口，以及行数据的结构，后面对于各种不同的文件，在实现时再进行适配

这里以csv文本文件为例，因为它有像Excel一样描述表格数据的能力，一般以\t分割行数据中的字段。由于它是纯文本文件，因此可以很容易的实现上面的接口，即将行读取操作委托给BufferedReader，然后再用给定的分隔符对行内容进行分割

对于行数据中的字段，可以简单地使用List保存

2.1.3. Excel类型识别

Excel文件分为xls和xlsx两种版本，并且它们的读取方式完全不同，但解析时面向的是文件流，无法判断是哪种版本，所以只好将判断Excel类型的任务交给使用者。

2.1.4. 顺序读取

使用者可能希望能够像读取下一行数据一样读取下一个sheet的数据，而且作为实现不能假设使用者的调用行为，他有可能穿插着调用readRow()和readSheet()。

引言中说过希望能像流水一样读取数据，所以不管是读取下一行，还是下一个sheet，只要没到最后，就可以一直读取下去，并且读过的数据不应该被再次读取。因此，为了避免读取的混乱，需要规定下readSheet()的语义：如果当前sheet已经读取过一些行并且还有剩余，那么直接返回当前sheet剩余的行，否则直接返回下一个sheet的所有行，这样比较符合流水读取的语义。

2.2. 实现

事实上，poi在Workbook初始化的时候，已经把整个Excel都解析结束并保存在内存中，这里所做的只是在它的结果基础上提供一些方法，方便获取数据。至于思路可以概括如下：

  1. 在Workbook初始化后维护一个Excel的sheet列表sheetNameList，并保持其顺序，另外维护一个sheet名称与数据的映射sheetMap

  2. 初始化一个sheetNameGivenList列表，后面sheet过滤时以及数据读取时都使用sheetNameGivenList，而sheetNameList保持不变，仅用作参考。比如读取时，首先从sheetNameGivenList中获取下一个要读取的sheet名称，然后再从sheetNameList中获取它真正的索引sheetIndex。

  3. 维护一些表示当前读取位置的索引，比如sheetIndexReading表示当前读取的sheet在sheetNameGivenList中的索引，由此也就知道了当前sheet的sheetName和sheetIndex，另外rowIndex表示当前行索引，cellIndex表示当前列索引。

  4. 读取流程就是从给定的sheetNameGivenList依次获取sheet，当一个sheet的最后一行读完时，就获取下一个sheet，直至最后一个sheet的最后一行读完时，返回null。

3. 示例

使用readRow()或者readSheet()可以像读取文本行一样读取Excel，这样使用者可以直接面向数据，而不用去在意Excel的结构或读取的状态。

4. 存在问题：内存消耗

由于poi在初始化时将整个Excel都解析完成并保存在内存中，而且为了保持数据的顺序使用了TreeMap进行层层嵌套，另外除了数据本身之外，还维护了单元格的各种属性，因此对内存的占用相当大，这样很容易造成内存溢出。下面以xlsx的实现XSSF为例，看一下数据的保存：

首先在XSSFWorkbook中维护着book信息，并将所有的sheet数据维护在一个List中

然后在XSSFSheet中，将所有的行数据维护在一个TreeMap中

接着在XSSFRow中，继续将所有的列数据嵌套在TreeMap中

最后在XSSFCell中实际保存单元格数据的元素是CTCell，其中保存了单元格的数据和所有属性。

为了避免内存溢出问题，poi另外也提供了基于SAX事件的读取方式，即不再一次性将Excel解析到内存中，而是在读取文件流的过程中，每当读取到一个节点就产生一个对应的事件，并交给调用者处理。SAX其实是java针对xml标准提供的基于事件解析的api，之所以能用来解析Excel，是因为Excel文件的本质上是一个压缩文件，其中包含了XML、图片及其他描述等文件，而其数据就是通过Xml来描述的。可以将文件后缀改为zip，然后使用压缩软件查看其内部文件结构
