根据设计文档中描述的设计和实现，这里提供了一些常见场景下的使用示例

1. 依赖

这里springboot中使用为例，添加如下依赖即可。只是目前spring-fom还没有提交到Maven仓库，需要自行将spring-fom打到本地仓库

2. 声明使用 @EnableFom

直接通过@EnableFom声明即可，然后spring会帮忙加载所有@FomSchedul标识的类

3. 定义任务 @FomSchedul

3.1. SimpleSchedule 定时单任务

对于简单的定时任务场景，没什么好讲的，所以这里顺便演示了一下，对于任务配置实时修改的支持。

示例中，对于@Value中的配置项email.user和email.address可以在界面实时修改合注入，而且在任务的执行过程中，还可以手动新增配置项dynamic.sleep.time，同样也能在界面查看和实时修改。

另外，对于定时计划的获取有一个优先级，即如果@FomSchedule中指定了，则以@FomSchedule中的定时计划为准，否则再尝试从@Scheduled中获取。

3.2. MultiSchedule 定时多任务

如果存在多个异构任务，但是定时计划相同，那么可以使用多个@Scheduled进行任务标识

对于定时计划的获取，同样如果@FomSchedule中指定了，则统一以@FomSchedule中为准；否则再尝试从下面的@Scheduled中获取，以获取到的第一个为准

3.3. BatchSchedule 定时批任务

如果希望定时执行一批任务，并且这些任务可以进行统一抽象，那么可以实现接口ScheduleFactory，以集合的方式提交一批任务， 但是对于任务的抽象需要继承给定的Task。另外，通过threadCore可以指定线程数来并行执行。

3.4. CompleteSchedule 自定义任务结束处理

如果希望在任务全部结束时，执行一些自定义操作，那么可以实现接口ScheduleCompleter。这在multi多任务或者batch批任务场景中比较有用，可以在最后一个任务结束时执行一些自定义的操作， 并且在执行中可以拿到所有任务的结果，执行的次数、以及本次执行开始时间等信息。

3.5. TimeoutSchedule 任务超时中断

如果想给任务指定超时时间，并在超时后进行取消，那么可以通过taskOverTime给任务指定超时时间，这样当任务超时后会尝试对其进行中断取消。

默认会对每个任务单独进行超时检测，如果希望对整体任务进行超时检测，可以通过detectTimeoutOnEachTask = false进行设置。这样将从第一个任务开始执行时计算超时时间，等到超时后就取消还没结束的任务， 而不管这个任务实际耗时多久，就算它还没开始执行（耗时 0ms）也会被取消。

3.6. CancelSchedule 自定义任务取消策略

如果想取消不可中断的任务，比如等待套接字返回，或者无限循环，那么此时线程已经无法响应中断，只能通过关闭套接字连接，或者增加循环结束标志等手段来实现。

这时，可以通过实现接口TaskCancelHandler来自定义任务取消策略，对于取消策略有两个调用时机：检测到任务超时，或者被外部shutdown关闭

如果想自定义shutdown之后的处理，比如释放一些资源，那么可以实现接口ScheduleTerminator。另外，通过execOnLoad可以设置任务在启动时立即执行。

4. 其它

虽然spring-fom主要用于计划任务场景，但并不要求@FomSchedule必须设置定时计划。如果没有定时计划，也可以将对应的ScheduleContext当成一个线程池来使用。 同样也有任务统计功能，并可以通过覆盖record(Result<E> result)方法来自定义任务结果统计，比如

然后使用的话就当成正常的线程池使用即可，但是对于提交的任务要求通过Task来封装

5. 界面接口支持

默认会提供一个内置的任务管理界面：http://{ip}:{port}/{path}/fom.html

当然，如果觉得界面不满足，或者希望将一些入口放到自己的界面中，那么也提供了对应的接口服务FomService，fom.html界面中所有的后台接口都是由FomService提供的。

如果觉得FomService提供的接口也不满足，那么可以通过注入所有的ScheduleContext来定制一些自己的接口
