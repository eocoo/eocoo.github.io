java容器Api[1.8].ArrayList与LinkedList

引言

java中的容器就是对各种数据结构的实现，按照接口实现可以分为Collection和Map。也可以按照是否线程安全来进行划分，比如JDK1.5之后，在concurrent包下面加了很多线程安全的容器， 具体放在并发工具介绍过之后在说明，因为它们的实现中或多或少都会依赖一些并发api。因此，接下来对于java容器，只介绍一些常见的基本容器实现，它们对于并发修改统一提供了一个及时反馈： ConcurrentModificationException，通过记一个修改次数来实现的，比如在迭代期间如果发现容器的修改次数与迭代开始时不一样，那么就立即失败并反馈异常给调用者， 事实上这个异常在单线程环境下也会触发，比如在迭代期间通过迭代器自身之外的方法修改容器，不管修改是不是来自同一个线程，因此需要注意。
另外，对于Hashtable或者Vector这种在JDK1.0实现的，已经被淘汰的容器也不说了，它们实现的想法也比较简单，就是通过一个内置锁限制所有对容器的访问和操作。

1. 概述

ArrayList与LinkedList都是线性数据结构，只不过一个数组，一个是链表。

  • 数据结构. 链表、队列、栈

数组的好处在于有位置索引，可以按下标任意访问，并且由于数组在内存上是连续的，因此访问会更快，但前提是元素必须排列紧凑，这样在删除元素时就免不了要进行移动， 即保证元素在位置上是连续的，不能有空缺，另外由于数组长度不可变，在添加元素也可能需要扩容。

链表可以弥补数组实现在插入删除上的缺陷，但代价是牺牲查询的效率。尤其是按位置索引进行访问时，它只能从前向后依次计数下标，或者反过来。

主要说一下add和remove以及迭代器的实现，其他操作都可以在此基础上进行实现，

2.3.1. add

add的实现主要是看它如何进行数组扩容的，它在达到DEFAULT_CAPACITY之后，每次会扩为原来的1.5倍。另外，实现中所有数组拷贝都是调用的System.arraycopy，它是一个Native方法， 考虑到效率问题，应该是C++实现的，因为C++能够直接操作到内存，而数组在内存上是连续的，且每个元素的占用长度确定，因此可以直接将数组的拷贝转成一次性内存拷贝。

2.3.2. remove

remove可以通过下标索引index来删除，也可以通过给定对象来删除所有与它equals的元素，注意的是删除所有而不是只删除第一个equals的元素。

2.3.3. iterator

ArrayList的iterator返回一个迭代器的私有实现Itr，称为迭代器模式，它的好处是封装和统一了对容器的迭代操作。另外，List的实现还都会返回一个扩展版的ListIterator，实现上是类似的。

3. LinkedList

LinkedList通过引用将元素链接成一个链表，理论上对于节点的数量没有限制的，不过由于size的类型为int，因此它的元素总数最多也为int最大值。 在实现时，它只需要记住两端的节点，即first和last，然后所有的操作都可以从头或者尾开始。

要说一下的是LinkedList除了是个List之外，它还实现了Deque，因此，在非线程安全的环境下，完全可以将LinkedList当成一个队列或者双端队列来使用， 就是java类库将列表和队列的链表实现放在一个类中了，另外也有基于数组实现的队列ArrayDeque，只是一般都很少用到，通常都能使用List代替，因此很少提及。

3.3.1. add

LinkedList的插入就是一个修改链接的过程，不过，最好不要指定位置插入，因为没有位置索引。

3.3.2. remove

remove的过程就是找到要删除的目标节点，然后将它从引用链中移除。
